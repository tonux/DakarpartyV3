Contribuer aux Guides Rails

Les Guides Rails visent à améliorer la documentation de Rails et à la rendre accessible à tous. Un développeur raisonnablement expérimenté devrait être capable d’utiliser les Guides pour monter rapidement en puissance sur Rails. Vous pouvez suivre l’effort global sur le Lighthouse des Guides Rails. Nos sponsors ont offerts des prix pour ceux qui écrivent un guide entier, mais il y a beaucoup d’autres façon de contribuer.

1 Comment Contribuer ?

  • Nous avons une politique d’engagement ouverte : tout le monde est bienvenu pour contribuer et corriger des contributions.
  • docrails est hébergé sur GitHub et a un droit d’écriture public.
  • Les Guides sont écrit en Textile, se trouvent dans le répertoire railties/guides/source du projet docrails.
  • Suivez les Conventions des Guides Rails.
  • Les ressources statiques sont dans le répertoire railties/guides/assets.
  • Exemple de format : Associations Active Record.
  • Résultat : Associations Active Record.
  • Vous pouvez générer les Guides en phase de test en exécutant rake generate_guides dans le répertoire railties.
  • Nous vous encourageons à valider votre XHTML pour les guides générés avant d’envoyer vos changements en exécutant rake validate_guides dans le répertoire railties.
  • Les derniers guides peuvent être consultés en lignes. Ce site web est généré périodiquement depuis docrails.

2 A Quoi Contribuer ?

  • Nous avons besoin d’auteurs, d’éditeurs, de correcteurs, et de traducteurs. Ajouter un seul paragraphe de contenu de qualité à in guide est un bon moyen de commencer.
  • Le moyen le plus facile de commencer est d’améliorer un guide existant:
    • Améliorez la structure pour la rendre cohérente.
    • Ajoutez des informations manquantes.
    • Corrigez toute erreur factuelle.
    • Rétablir les typos ou améliorer le style.
    • Mettez à jour avec la dernière version de Edge Rails.
  • Nous sommes également ouvert aux suggestions pour de tout nouveau guides:
    • Contactez lifo ou fxn pour voir votre idée approuvée. Voyez la section Contact ci-dessous
    • Si vous êtes l’auteur principal d’un guide significatif, vous êtes éligible pour les prix.

3 Quel est le processus?

  • Le moyen de contribuer préféré est de commiter directement sur docrails.
  • Un nouveau guide n’est édité par son auteur qu’une fois qu’il est terminé. Dans ce cas, un retour peu être donné dans son ticket Lighthouse.
  • Si vous écrivez un nouveau guide envoyez librement votre travail sur docrails et avertissez lifo ou fxn quand vous avez terminé un premier brouillon.
  • Les correcteurs des Guides fourniront ensuite un retour, certains possiblement sous la forme de commit direct afin d’agiliser le processus.
  • Finalement le guide sera approuvé et ajouté à l’index.

4 Prix

Pour chaque guide complété, le contributeur principal recevra tout les prix suivants:

  • $200 du Caboose Rails Documentation Project.
  • 1 an du plan Micro GitHub à $84.
  • 1 an de RPM Basic (Production performance management) pour plus de 10 serveur valant 12 mois x $40 par serveur x 10 serveur = $4800. Et également, une économie de $45 par serveur par mois sur une liste de prix pour monter vers des produits avancés.

5 Règles

  • Les Guides sont sous licence Creative Commons Attribution-Share Alike 3.0.
  • Si vous n’êtes pas sur de quel guide est actuellement en cours de travail, commencez par IRC et demandez.
  • Si le même auteur veux écrire plusieurs guides, c’est idéalement la situation dans laquelle nous aimerions être! Cependant, cet auteur ne recevrait que le prix en argents comptant pour chacun des guides (et pas le GitHub ou les prix RPM).
  • Notre équipe de correction aura le dernier mot sur le fait qu’un guide soit complet et suffisamment de bonne qualité.

Tous les auteurs devraient lire et suivre les Conventions des Guides Rails et les Conventions de la Documentation d’API Rails.

6 Traductions

L’effort de traduction pour les Guides Rails viens de commencer. Nous sommes au courant de projets de traduction des Guides en Espagnol, Portugais, Polonais, et Français. Pour plus de détails ou pour participer voyez le page des "Guides de Traduction Rails ":http://wiki.github.com/lifo/docrails/translating-rails-guides.

Feedback

Vous êtes encouragé à aider à maintenir la qualité de ce guide.

Si vous voyez des fautes ou des erreurs factuelles qui vous semblent faciles à corriger, n'hésitez pas à cloner docrails et à pousser les modifications vous-même. Cette branche de Rails a un accès public en écriture. Les commits sont toujours revus mais cela se produit après la soumission de votre contribution. docrails est "cross-merged" (fusionné) avec la branche master régulièrement.

Il se peut que vous trouviez également du contenu incomplet, ou obsolète. Vous pouvez ajoutez la documentation manquante à la branche master. Consultez les consignes pour les guides Ruby on Rails pour le style d'écriture et les conventions.

Les problèmes peuvent être rapportés dans Github.

Enfin, mais c'est important, toute discussion à propos de documentation Ruby on Rails est bienvenue dans la mailing list rubyonrails-docs.
